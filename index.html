Decomposing git init

Makes the current directory int a git repository. To do this, it creates a .git directory and writes some files to it. They define configuration and history. They are editable.

Decomposing git add

This has two effects. It creates a blob file that contains the compressed content of the files added. It does this by using a hashing algorithm. Second, it adds the file to the index. The index is a list that contains of every file Git has been told to keep track of. When you make a change to a file, and run git add you will create a new blob.

Decomposing git commit

$git commit -m 'first commit'

This commit is called 'first commit'. The git command has three steps: it creates a tree graph for the content of the version of the project that is about to be committed. It creates a commit object. It points the current branch at the new commit object.

Create a tree graph

Git records the state of the project by creating a tree graph from the index. The graph is composed of two types of objects: blobs and trees. Blobs are stored by git add. Trees are stored when a commit is made. A tree represents a directory in the working copy.

Operating Systems: Part 2 Processes

October 23, 2016 by Tim

A process is a software object that represenets a running program. An object is a contains data and provides methods that operate on data. A process is an object that contains the following data:

  • The text of a program, usually a sequence of machine language instructions.
  • Data associated with the program, including static data (allocated at compile time) and dynamic data including the run time stack and the heap.
  • The state if any pending input/output operations. For example, if the process is waiting for data to be read from disk or for a packet to arrive on a network, the status of these operations is part of the process.
  • The hardware state of the program, which includes data stored in registers, the status information, and the program counter, which indicates which instruction is currently executing.

Operating Systems: Under the Hood Part 1

October 23, 2016 by Tim

People often tend to think of programming languages as either compiled or interpreted. But the distinction is not always as clear cut.

First, many languages can be either compiled or interpreted. There are C interpreters and Python compilers. Java uses a hybrid approach, compiling programs into an intermediate language and then running the translated program in an interpreter. The intermediate language is called Java bytecode, which is similar to machine language , but is executed by a software interpreter, the Java Virtual Machine.

This is the C complilation process:

  • Preprocessing: C includes "preprocessor directives" that take effect before the program is compiled. For example #include directive causes the source code from another file to be inserted at the location of the directive.
  • Parsing: During parsing, the compiler reads the source code and builds an internal representation of the program, called an "abstract syntax tree." Errors detected at this step are usually sytax errors.
  • Static Checking: The compiler checks whether variables and values have the right type, whether functions are called with the right number and type of arguments. Errors detected during this step are sometimes called "static semantic" errors.
  • Code generation: The compiler reads the internal representation of the program and develops bytecode
  • Linking: If the program uses values and functions defined in a library, the compiler has to find the appropriate library and include the required code.
  • Optimization: At several points in the process, the compiler can transform the program to generate code that runs faster or uses less space. Most optimizations are simple changes that eliminate obvious waste, but some compilers perform sophisticated analysis and transformations.

Object Oriented Design Part 1

October 23, 2016 by Tim

Class versus Interface Inheretence. An object's class defines how the object is to be implemented. The class determines the objects internal state and the imlementation of its operations.In contrast the objects type refers to interface-the set of requests to which it can respond. An object can have many types, and objects of different classes can have the same type.

Interface versus implementation. When inheritance is used carefully, all classes derived from an abstract class will share its interface. This implies that a subclass merely adds or overrides operations and does not hide operations of the parent class. All subclasses can then respond to the requests in the interface of this abstract class, making them all subtypes of the abstract class.

This greatly reduces implementation dependencies between subsystems that it leads to the following principle of reusable object-oriented design: Program to interface, not an implementation.

Inheritance versus Composition. The two most common techniques for reusing functionality in object-oriented systems are class inheritance and object composition. As we've explained, inheritance lets you define the implementationof one class in terms of another's. Reuse by subclassing is often referred to white box reuse.

Object composititon is an alternative to class inheritance. Here, new functionality is obtained by assembling or composing objects to get more complex functionality. Object composition requires that the objects being composed have well-defined interfaces. This style of reuse is called black-box reuse, because no details of an object are visible.

Inheritance and composition each have their advantages and disadavntages. Class inheritence is defined statically at compile-time and is straightforward to use, since it's supported directly by the programming language.Class inheritance also makes it easier to modify the implementation being reused. When a subclass overrides some but not all operations, it can affect the operations it inherits as well, assuming they call the overidden operations. But class inheritence has some disadvantages, too. First you can't choose the implementations inherited for the parent at run-time, because inheritence is defined at compile-time. Second, and generally worse, parent classes often define at least compile-time. Second, and generally worse, parent classes often define at least part of their subclasses physical representation. Because inheritance exposes a subclass to details of its parent's implementation, it's often said that "inheritence break encapsulation". The implementation of a subclass bcomes so bound up with the implementation of its parent class that any change in the parent's implementation will force the subclass to change,

Inheritance vs. Composition. The Gang of Four tell us to value object composition over inheritance. It's their opinion that many developers overuse inheritance leading to large unstable systems. Think carefully about the composition of your classes and take care to see that the relationships between subclasses are few but strong.

Delegation. In delegation two objects are used in handling a request: a receiving object delegate operations to its delegate.This is analagous to subclasses defering requests to its parent classes. But with inheretence, an inherited operation can always refer to the receiving variable as self. To achieve the same in delegation, the reciver passes itself to the delegate to let the delegated operation refer to the reciever.

October 22, 2016 by Tim

Dulles arrives in Bern in early 1942. All of Europe is occupied by the Axis, and Stalingrad has not yet turned the tide of the war on the Eastern Front. Dulles is behind enemy lines in Switzerland, however in some way he's at home. He worked in Bern for the State Department during the First World War and his company did business in Berlin.

Talbot, the author of the book, accuses Dulles of using his role during the war to protect his corporate clients with holdings in Europe. Foster at this time was in New York at Sullivan and Cromwell hiding the U.S. assets of IG Farben and Merck KGaA.. As both Dulles advise large corporations in New York, they probably see the encroachment of the Soviet Union and the march of the Red Army west as the true enemy to its client's assets, as the Soviet Union will likely confiscate property. Talbot states that the Dulles brothers used financing from American companies like General Motors, General Electric, and DuPot Chemical to invest in German industrial companies like IG Farben and Volkwagon during the war. Per Talbot, Dulles sees his role in the OSS as an opportunity to help his corporate clients in the United States, and is distrustful of what will happen when a peace settlment is made with the Soviet Union.

In March of 1945, Dulles makes a peace agreement with Waffen-SS General Wolff to hand over Northern Italy to the United States. This happens two months before VE day, and prevents Northern Italy from coming under communist control. Immediately after the war this becomes the border of the iron curtain, west Germany and Italy become part of NATO, while Austria and East Germany become satellites of the Soviet Union. In return for surrender his surrender, Dulles promises to keep General Wolff from a trial in Nuremberg.
